THREADS LAB

개인정보처리방침

시행일: 2026년 7월 19일

요약: Threads Lab은 사용자가 명시적으로 승인한 Threads API 데이터를 개인용 연구 목적으로 처리하는 로컬 도구입니다. 자격증명과 수집 결과를 운영자의 Mac 밖에 별도로 저장하거나 판매하지 않습니다.

1. 서비스와 운영자

Threads Lab은 Threads의 관련 대화 후보를 검토하기 위해 Seonghwan Oh가 운영하는 개인용 연구 도구입니다. 이 방침은 Threads Lab이 Meta Threads API를 통해 처리하는 정보에 적용됩니다.

2. 처리하는 정보

  • OAuth를 승인한 Threads 계정의 사용자 ID와 사용자 이름
  • 검색어와 검색 실행 시각
  • Threads API가 반환한 게시물 ID, 본문, 대체 텍스트, 게시 시각, 영구 링크, 미디어 유형, 주제 태그와 공개 반응 지표
  • 운영자가 기록한 후보 게시물의 보관·제외 결정과 메모

Threads Lab은 비밀번호를 수집하지 않으며, 승인된 API 범위를 벗어난 비공개 Threads 정보에 접근하지 않습니다.

3. 이용 목적

정보는 관련 대화의 흐름과 표현 방식을 연구하고, 사람이 검수할 비공개 후보 목록을 만들며, API 연결 상태를 점검하는 데만 사용됩니다. 현재 Threads Lab은 게시물을 자동으로 게시하지 않습니다.

4. 저장 위치와 보유 기간

API 토큰과 앱 자격증명은 운영자의 macOS Keychain에 저장됩니다. 수집 결과와 검수 기록은 운영자의 Mac에 있는 비공개 SQLite 데이터베이스와 로컬 문서에 저장됩니다. 게시물 원문 필드는 수집 시점부터 30일의 만료 시점을 가지며 정리 작업에서 제거됩니다. 나머지 메타데이터와 검수 기록은 연구 목적에 필요한 기간 동안 보유한 뒤 로컬 데이터베이스와 백업에서 삭제합니다.

5. 외부 제공과 판매

Threads Lab은 개인정보를 판매하거나 광고 목적으로 공유하지 않습니다. 인증과 데이터 조회 과정에서 Meta의 Threads API와 통신하는 것 외에는, 현재 수집 데이터를 외부 분석·광고 서비스나 별도의 운영 서버로 전송하지 않습니다.

6. 보호 조치

자격증명은 소스 코드와 데이터베이스에 기록하지 않고 macOS Keychain으로 분리합니다. 수집 데이터가 있는 디렉터리는 공개 저장소에서 제외하며, 필요한 최소 API 권한만 요청합니다.

7. 승인 철회와 데이터 삭제

사용자는 Threads의 웹사이트 권한 설정에서 언제든 앱 승인을 철회할 수 있습니다. 삭제를 요청하면 해당 사용자와 연결된 OAuth 자격증명, 로컬 수집 결과와 검수 기록을 확인하여 삭제합니다. 요청하려면 아래 이메일로 Threads 사용자 이름과 함께 연락해 주세요.

8. 방침 변경

LLM 분석, 외부 서버, 자동 게시 등 데이터 처리 방식이 추가되면 적용 전에 이 방침을 갱신하고 시행일을 표시합니다.
